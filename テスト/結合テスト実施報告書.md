## 4. 失敗テスト詳細と根本原因分析

### 4.1 外部API認証問題（高優先度）
**影響テスト**: OpenAI統合、Graph API統合関連テスト
**根本原因**: 
- OpenAI API キーが無効（401エラー）
- Microsoft Graph API認証情報未設定（503エラー）

**失敗例**:
```
ERROR: OpenAI API error: Error code: 401 - {'error': {'message': 'Incorrect API key provided'}}
```

### 4.2 スキーマ検証エラー（中優先度）
**影響テスト**: データ整合性、データベーストランザクション関連テスト
**根本原因**: 
- 必須フィールドの不足（422 Unprocessable Entity）
- リクエストデータの形式不整合

**失敗例**:
```
assert 422 == 201  # 期待: 201 Created, 実際: 422 Unprocessable Entity
```

### 4.3 API レスポンス形式不整合（中優先度）
**影響テスト**: セキュリティ統合、パフォーマンス統合テスト
**根本原因**: 
- テストコードでの不正なJSONキーアクセス
- APIレスポンス構造の理解不足

**失敗例**:
```
KeyError: 'access_token'
TypeError: list indices must be integers or slices, not str
```

### 4.4 HTTPステータスコード不整合（低優先度）
**影響テスト**: セキュリティテスト
**根本原因**: 
- 期待値と実際の応答コードの相違（401 vs 403）

**失敗例**:
```
assert 403 == 401  # 期待: 401 Unauthorized, 実際: 403 Forbidden
```

### 4.5 パフォーマンス閾値未達成（低優先度）
**影響テスト**: パフォーマンス統合テスト
**根本原因**: 
- 並行処理性能の期待値設定が過大
- データベースクエリ性能の測定方法に問題
