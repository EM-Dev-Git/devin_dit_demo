# トランスクリプトから議事録作成API 実装計画書

## 1. プロジェクト概要

### 目的
音声トランスクリプトを入力として、OpenAIを活用して構造化された議事録を自動生成するAPIシステムの構築

### 主要機能
- トランスクリプト入力による議事録自動生成
- JWTトークンベースの認証システム
- ユーザー管理機能
- ログ機能
- 環境変数による設定管理

## 4. API設計

### 認証エンドポイント
```
POST /auth/register    # ユーザー登録
POST /auth/login       # ログイン（JWTトークン発行）
POST /auth/refresh     # トークンリフレッシュ
```

### 議事録生成エンドポイント
```
POST /minutes/generate # トランスクリプトから議事録生成
GET  /minutes/history  # 生成履歴取得
GET  /minutes/{id}     # 特定議事録取得
```

### ユーザー管理エンドポイント
```
GET  /users/profile    # プロフィール取得
PUT  /users/profile    # プロフィール更新
```

## 5. データベース設計

### Usersテーブル
```sql
CREATE TABLE users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    username VARCHAR(50) UNIQUE NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    hashed_password VARCHAR(255) NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### Minutesテーブル
```sql
CREATE TABLE minutes (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,
    title VARCHAR(200),
    transcript TEXT NOT NULL,
    generated_minutes TEXT NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users (id)
);
```

## 6. 認証システム設計

### JWT実装方針
- **アルゴリズム**: HS256
- **トークン有効期限**: 24時間
- **リフレッシュトークン**: 7日間
- **パスワードハッシュ**: bcrypt

### セキュリティ要件
- パスワード強度チェック
- レート制限
- CORS設定
- セキュアヘッダー設定

## 7. OpenAI連携設計

### プロンプト設計
```python
MINUTES_PROMPT = """
以下のトランスクリプトを基に、構造化された議事録を作成してください。

【出力形式】
# 会議議事録
## 日時・参加者
## 議題
## 決定事項
## アクションアイテム
## 次回予定

【トランスクリプト】
{transcript}
"""
```

### API呼び出し設計
- **モデル**: gpt-3.5-turbo または gpt-4
- **最大トークン数**: 2000
- **温度**: 0.3（一貫性重視）
- **エラーハンドリング**: リトライ機能付き

## 8. ログ設計

### ログレベル
- **DEBUG**: 開発時デバッグ情報
- **INFO**: 一般的な処理情報
- **WARNING**: 警告レベルの問題
- **ERROR**: エラー情報
- **CRITICAL**: システム停止レベルの問題

### ログ出力項目
- タイムスタンプ
- ログレベル
- ユーザーID（認証済みの場合）
- エンドポイント
- 処理時間
- エラー詳細

