# トランスクリプトから議事録作成API 実装計画書

## 1. プロジェクト概要

### 1.1 目的
音声トランスクリプトから議事録を自動生成するAPIシステムの構築

### 1.2 主要機能
- トランスクリプトから議事録への変換（OpenAI API使用）
- JWT認証によるユーザー認証システム
- ユーザー管理（DB保存）
- ログ機能
- 環境変数による設定管理

### 1.3 対象ユーザー
- 会議の議事録作成を効率化したい組織
- 音声データから構造化された議事録を必要とするチーム
- API経由で議事録生成機能を利用したいシステム開発者

## 2. システム構成

### 2.1 ディレクトリ構造
```
project/
├── routers/           # APIルーティング
│   ├── __init__.py
│   ├── auth.py        # 認証関連エンドポイント

│   ├── openai_client.py # OpenAI API連携
│   └── logger.py      # ログ機能
```

### 2.2 技術スタック
- **フレームワーク**: FastAPI
- **認証**: JWT (JSON Web Token)
- **データベース**: SQLite（開発用）/ PostgreSQL（本番用）
- **AI API**: OpenAI GPT API
- **ログ**: Python標準ログライブラリ
- **環境管理**: python-dotenv
- **バリデーション**: Pydantic
- **ORM**: SQLAlchemy

## 4. API設計

### 4.1 認証エンドポイント (routers/auth.py)
```
POST /auth/register     # ユーザー登録
POST /auth/login        # ログイン
POST /auth/refresh      # トークンリフレッシュ
GET  /auth/me          # 現在のユーザー情報取得
```

### 4.2 議事録エンドポイント (routers/minutes.py)
```
POST /minutes/generate  # トランスクリプトから議事録生成
GET  /minutes/{id}     # 議事録取得
GET  /minutes/         # ユーザーの議事録一覧
PUT  /minutes/{id}     # 議事録更新
DELETE /minutes/{id}   # 議事録削除
```

### 4.3 ユーザー管理エンドポイント (routers/users.py)
```
GET  /users/me         # 現在のユーザー情報
PUT  /users/me         # ユーザー情報更新
DELETE /users/me       # アカウント削除
```

### 4.4 ヘルスチェック
```
GET  /health          # システム状態確認
GET  /               # API情報
```

## 5. データベース設計

### 5.1 Userテーブル
```sql
CREATE TABLE users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    username VARCHAR(50) UNIQUE NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    is_active BOOLEAN DEFAULT TRUE
);
```

### 5.2 Minutesテーブル
```sql
CREATE TABLE minutes (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,
    meeting_title VARCHAR(200) NOT NULL,
    transcript TEXT NOT NULL,
    summary TEXT NOT NULL,
    key_points TEXT NOT NULL,
    action_items TEXT NOT NULL,
    participants TEXT NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users (id)
);
```

## 8. ログ設計

### 8.1 ログレベル
- **ERROR**: システムエラー、API呼び出し失敗
- **WARNING**: 認証失敗、レート制限
- **INFO**: API呼び出し、ユーザー操作
- **DEBUG**: 詳細なデバッグ情報

### 8.2 ログ形式
```json
{
    "timestamp": "2024-01-01T12:00:00Z",
    "level": "INFO",
    "message": "User login successful",
    "user_id": 123,
    "endpoint": "/auth/login",
    "ip_address": "192.168.1.1",
    "processing_time": 0.15
}
```

### 8.3 ログ出力先
- 開発環境: コンソール + ファイル
- 本番環境: ファイル + 外部ログサービス
