## 3. 物理テーブル設計

### 3.1 Users テーブル

#### DDL定義
```sql
CREATE TABLE users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    username VARCHAR(50) NOT NULL UNIQUE,
    email VARCHAR(100) NOT NULL UNIQUE,
    hashed_password VARCHAR(255) NOT NULL,
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);
```

#### SQLAlchemy モデル定義
```python
from sqlalchemy import Column, Integer, String, DateTime, Text
from sqlalchemy.ext.declarative import declarative_base
from datetime import datetime

Base = declarative_base()

class User(Base):
    __tablename__ = "users"
    
    id = Column(Integer, primary_key=True, index=True, autoincrement=True)
    username = Column(String(50), unique=True, index=True, nullable=False)
    email = Column(String(100), unique=True, index=True, nullable=False)
    hashed_password = Column(String(255), nullable=False)
    created_at = Column(DateTime, default=datetime.utcnow, nullable=False)
    updated_at = Column(DateTime, default=datetime.utcnow, onupdate=datetime.utcnow, nullable=False)
    
    # リレーションシップ
    minutes = relationship("Minutes", back_populates="user", cascade="all, delete-orphan")
```

#### 物理属性詳細
| 属性名 | データ型 | 長さ | NULL | デフォルト | インデックス | 説明 |
|--------|----------|------|------|-----------|-------------|------|
| id | INTEGER | - | NO | AUTOINCREMENT | PRIMARY | 主キー、自動増分 |
| username | VARCHAR | 50 | NO | - | UNIQUE | ユーザー名、一意インデックス |
| email | VARCHAR | 100 | NO | - | UNIQUE | メールアドレス、一意インデックス |
| hashed_password | VARCHAR | 255 | NO | - | - | bcryptハッシュ（60文字+余裕） |
| created_at | TIMESTAMP | - | NO | CURRENT_TIMESTAMP | - | 作成日時 |
| updated_at | TIMESTAMP | - | NO | CURRENT_TIMESTAMP | - | 更新日時 |

### 3.2 Minutes テーブル

#### DDL定義
```sql
CREATE TABLE minutes (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,
    title VARCHAR(200),
    transcript TEXT NOT NULL,
    generated_minutes TEXT NOT NULL,
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users (id) ON DELETE CASCADE
);
```

#### SQLAlchemy モデル定義
```python
class Minutes(Base):
    __tablename__ = "minutes"
    
    id = Column(Integer, primary_key=True, index=True, autoincrement=True)
    user_id = Column(Integer, ForeignKey("users.id"), nullable=False, index=True)
    title = Column(String(200), nullable=True)
    transcript = Column(Text, nullable=False)
    generated_minutes = Column(Text, nullable=False)
    created_at = Column(DateTime, default=datetime.utcnow, nullable=False, index=True)
    
    # リレーションシップ
    user = relationship("User", back_populates="minutes")
```

#### 物理属性詳細
| 属性名 | データ型 | 長さ | NULL | デフォルト | インデックス | 説明 |
|--------|----------|------|------|-----------|-------------|------|
| id | INTEGER | - | NO | AUTOINCREMENT | PRIMARY | 主キー、自動増分 |
| user_id | INTEGER | - | NO | - | FOREIGN, INDEX | 外部キー、インデックス |
| title | VARCHAR | 200 | YES | NULL | - | 会議タイトル |
| transcript | TEXT | - | NO | - | - | 元トランスクリプト |
| generated_minutes | TEXT | - | NO | - | - | 生成議事録 |
| created_at | TIMESTAMP | - | NO | CURRENT_TIMESTAMP | INDEX | 作成日時、インデックス |

#### ストレージ見積もり
```
1レコードあたりのサイズ（平均）:
- id: 8 bytes (INTEGER)
- user_id: 8 bytes (INTEGER)
- title: 50 bytes (平均)
- transcript: 1,000 bytes (平均)
- generated_minutes: 2,000 bytes (平均)
- created_at: 19 bytes (TIMESTAMP)
- 合計: 約3,085 bytes/レコード

1,000議事録の場合: 約3MB
```

### 8.2 トランザクション管理

#### トランザクション設計
```python
from sqlalchemy.orm import sessionmaker
from contextlib import contextmanager

SessionLocal = sessionmaker(autocommit=False, autoflush=False, bind=engine)

@contextmanager
def get_db_transaction():
    """トランザクション管理付きDBセッション"""
    db = SessionLocal()
    try:
        yield db
        db.commit()
    except Exception:
        db.rollback()
        raise
    finally:
        db.close()
```

```python
import time
import threading
from concurrent.futures import ThreadPoolExecutor

def performance_test():
    """データベースパフォーマンステスト"""
    
    def insert_test_data(thread_id, count):
        """テストデータの挿入"""
        with get_db_transaction() as db:
            for i in range(count):
                user = User(
                    username=f"test_user_{thread_id}_{i}",
                    email=f"test_{thread_id}_{i}@example.com",
                    hashed_password="hashed_password"
                )
                db.add(user)
    
    # 並行挿入テスト
    start_time = time.time()
    with ThreadPoolExecutor(max_workers=10) as executor:
        futures = [executor.submit(insert_test_data, i, 100) for i in range(10)]
        for future in futures:
            future.result()
    
    insert_time = time.time() - start_time
    print(f"Insert test completed in {insert_time:.2f} seconds")
```
