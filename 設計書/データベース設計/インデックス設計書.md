# インデックス設計書

## 1. 概要

### 目的
トランスクリプトから議事録作成APIシステムのデータベースインデックス設計を詳細に定義し、クエリパフォーマンスの最適化を実現する

### 対象範囲
- 全テーブルのインデックス設計
- クエリパフォーマンス最適化
- インデックス保守戦略
- パフォーマンス監視

### 前提条件
- SQLite データベースの使用
- 想定データ量とアクセスパターン
- システムリソース制約

## 2. 設計方針

### 基本方針
- **パフォーマンス優先**: 頻繁なクエリの高速化
- **ストレージ効率**: 必要最小限のインデックス
- **保守性**: 理解しやすいインデックス構成
- **拡張性**: 将来的なデータ増加に対応

### 制約事項
- SQLite のインデックス機能制限
- ストレージ容量の制約
- 更新性能への影響

### 品質要件
- **応答性**: クエリ応答時間の最適化
- **効率性**: インデックスサイズの最適化
- **可用性**: インデックス破損時の復旧

## 3. クエリ分析

#### 頻度別クエリ分類
| クエリタイプ | 頻度 | 重要度 | 説明 |
|-------------|------|--------|------|
| ユーザー認証 | 高 | 高 | ログイン時のユーザー検索 |

| 特定議事録取得 | 中 | 高 | 議事録詳細表示 |
| ユーザー情報取得 | 中 | 中 | プロフィール表示 |
| 議事録検索 | 低 | 中 | 日付範囲検索 |

## 4. インデックス設計

### 4.1 主キーインデックス

#### 自動作成インデックス
```sql
-- Users テーブル主キーインデックス
-- 自動作成: CREATE UNIQUE INDEX sqlite_autoindex_users_1 ON users(id)

-- Minutes テーブル主キーインデックス  
-- 自動作成: CREATE UNIQUE INDEX sqlite_autoindex_minutes_1 ON minutes(id)
```

#### 特徴
- **自動作成**: PRIMARY KEY 制約により自動生成
- **一意性**: 重複値なし
- **クラスタ化**: SQLiteでは主キーがクラスタ化インデックス
- **使用場面**: ID指定での単一レコード取得

### 4.2 一意制約インデックス

#### Users テーブル
```sql
-- ユーザー名一意インデックス
CREATE UNIQUE INDEX uk_users_username ON users (username);

-- メールアドレス一意インデックス
CREATE UNIQUE INDEX uk_users_email ON users (email);
```

#### インデックス詳細
| インデックス名 | 対象列 | タイプ | サイズ見積もり | 用途 |
|---------------|--------|--------|---------------|------|
| uk_users_username | username | UNIQUE | 50KB (1万ユーザー) | ログイン認証 |
| uk_users_email | email | UNIQUE | 100KB (1万ユーザー) | 重複チェック |

#### 使用クエリ例
```sql
-- uk_users_username 使用
SELECT id, username, email, hashed_password 
FROM users 
WHERE username = 'testuser';

-- uk_users_email 使用
SELECT id FROM users 
WHERE email = 'test@example.com';
```

### 4.3 外部キーインデックス

#### Minutes テーブル
```sql
-- user_id 外部キーインデックス
CREATE INDEX ix_minutes_user_id ON minutes (user_id);
```

#### インデックス詳細
| インデックス名 | 対象列 | タイプ | サイズ見積もり | 用途 |
|---------------|--------|--------|---------------|------|
| ix_minutes_user_id | user_id | NON-UNIQUE | 80KB (10万議事録) | ユーザー別検索 |

#### 使用クエリ例
```sql
-- ix_minutes_user_id 使用
SELECT * FROM minutes 
WHERE user_id = 1;

-- JOIN クエリでも使用
SELECT m.*, u.username 
FROM minutes m 
JOIN users u ON m.user_id = u.id 
WHERE u.username = 'testuser';
```

### 4.4 複合インデックス

#### 最適化対象クエリ
```sql
-- 最も頻繁なクエリパターン

FROM minutes 
WHERE user_id = ? 
ORDER BY created_at DESC
```

#### 複合インデックス設計

#### インデックス効果

```sql
-- インデックスなしの場合
```

## 5. インデックス実装

### 5.1 SQLAlchemy での実装

#### モデル定義でのインデックス指定
```python
from sqlalchemy import Column, Integer, String, DateTime, Text, Index
from sqlalchemy.ext.declarative import declarative_base

Base = declarative_base()

class User(Base):
    __tablename__ = "users"
    
    id = Column(Integer, primary_key=True, index=True)
    username = Column(String(50), unique=True, index=True, nullable=False)
    email = Column(String(100), unique=True, index=True, nullable=False)
    hashed_password = Column(String(255), nullable=False)
    created_at = Column(DateTime, default=datetime.utcnow, nullable=False)
    updated_at = Column(DateTime, default=datetime.utcnow, onupdate=datetime.utcnow, nullable=False)

class Minutes(Base):
    __tablename__ = "minutes"
    
    id = Column(Integer, primary_key=True, index=True)
    user_id = Column(Integer, ForeignKey("users.id"), nullable=False, index=True)
    title = Column(String(200), nullable=True)
    transcript = Column(Text, nullable=False)
    generated_minutes = Column(Text, nullable=False)
    created_at = Column(DateTime, default=datetime.utcnow, nullable=False, index=True)
    
    # 複合インデックスの定義
    __table_args__ = (
        Index('ix_minutes_user_created', 'user_id', 'created_at'),
    )
```

#### 明示的インデックス作成
```python
from sqlalchemy import create_engine, Index

# エンジン作成
engine = create_engine("sqlite:///./app.db")

# 複合インデックスの明示的作成
def create_indexes():
    with engine.connect() as conn:
        # 複合インデックス
        conn.execute("""
            CREATE INDEX IF NOT EXISTS ix_minutes_user_created 
            ON minutes (user_id, created_at DESC)
        """)
        
        # 日付インデックス
        conn.execute("""
            CREATE INDEX IF NOT EXISTS ix_minutes_created_at 
            ON minutes (created_at DESC)
        """)
        
        conn.commit()
```

```python
import sqlite3
from contextlib import contextmanager

@contextmanager
def measure_time(description):
    start = time.time()
    yield
    end = time.time()
    print(f"{description}: {end - start:.4f} seconds")

def benchmark_queries():
    conn = sqlite3.connect("app.db")
    
    # テストデータ準備（10万件の議事録）
    setup_test_data(conn)
    
    # 1. ユーザー認証クエリ
    with measure_time("User authentication"):
        for i in range(1000):
            cursor = conn.execute(
                "SELECT id, username, email, hashed_password FROM users WHERE username = ?",
                (f"user_{i % 100}",)
            )
            cursor.fetchone()
    
    # 2. 議事録一覧取得クエリ
    with measure_time("Minutes list query"):
        for i in range(1000):
            cursor = conn.execute(
                "SELECT id, title, created_at FROM minutes WHERE user_id = ? ORDER BY created_at DESC LIMIT 10",
                (i % 100 + 1,)
            )
            cursor.fetchall()
    
    # 3. 特定議事録取得クエリ
    with measure_time("Specific minutes query"):
        for i in range(1000):
            cursor = conn.execute(
                "SELECT * FROM minutes WHERE id = ? AND user_id = ?",
                (i % 10000 + 1, i % 100 + 1)
            )
            cursor.fetchone()
    
    conn.close()
```

## 7. インデックス保守

### 7.1 インデックス再構築

#### 定期的な再構築スクリプト
```python
def rebuild_indexes():
    """インデックスの再構築"""
    conn = sqlite3.connect("app.db")
    
    try:
        # 統計情報の更新
        conn.execute("ANALYZE")
        
        # インデックスの再構築
        conn.execute("REINDEX")
        
        print("Index rebuild completed successfully")
        
    except Exception as e:
        print(f"Index rebuild failed: {e}")
        raise
    finally:
        conn.close()

def rebuild_specific_index(index_name):
    """特定インデックスの再構築"""
    conn = sqlite3.connect("app.db")
    
    try:
        conn.execute(f"REINDEX {index_name}")
        print(f"Index {index_name} rebuilt successfully")
        
    except Exception as e:
        print(f"Failed to rebuild index {index_name}: {e}")
        raise
    finally:
        conn.close()
```

### 7.2 インデックス監視

#### インデックスサイズ監視
```python
def monitor_index_sizes():
    """インデックスサイズの監視"""
    conn = sqlite3.connect("app.db")
    
    # インデックス一覧の取得
    cursor = conn.execute("""
        SELECT name, sql 
        FROM sqlite_master 
        WHERE type = 'index' 
        AND tbl_name IN ('users', 'minutes')
        ORDER BY tbl_name, name
    """)
    
    indexes = cursor.fetchall()
    
    print("Index Information:")
    print("-" * 50)
    for name, sql in indexes:
        print(f"Index: {name}")
        print(f"SQL: {sql}")
        print("-" * 50)
    
    # データベース全体のサイズ
    cursor = conn.execute("PRAGMA page_count")
    page_count = cursor.fetchone()[0]
    
    cursor = conn.execute("PRAGMA page_size")
    page_size = cursor.fetchone()[0]
    
    total_size = page_count * page_size
    print(f"Total database size: {total_size:,} bytes ({total_size/1024/1024:.2f} MB)")
    
    conn.close()
```
