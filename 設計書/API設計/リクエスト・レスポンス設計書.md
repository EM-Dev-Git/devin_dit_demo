# リクエスト・レスポンス設計書

## 1. 概要

### 目的
トランスクリプトから議事録作成APIシステムのリクエスト・レスポンス形式を詳細に定義し、データ構造とバリデーションルールを明確化する

### 対象範囲
- 全APIエンドポイントのリクエスト・レスポンス仕様
- Pydantic スキーマ定義
- バリデーションルール
- データ変換処理

### 前提条件
- Pydantic による型安全なデータ検証
- JSON 形式でのデータ交換
- FastAPI の自動ドキュメント生成対応

## 2. 設計方針

### 基本方針
- **型安全性**: Pydantic による厳密な型検証
- **一貫性**: 統一されたデータ構造
- **拡張性**: 将来的なフィールド追加に対応
- **セキュリティ**: 機密情報の適切な処理

### 制約事項
- JSON シリアライゼーション可能なデータ型のみ
- ファイルサイズ制限の考慮
- 文字エンコーディング（UTF-8）

### 品質要件
- **バリデーション**: 入力値の厳密な検証
- **パフォーマンス**: 効率的なデータ変換
- **可読性**: 理解しやすいスキーマ定義

## 3. 認証系リクエスト・レスポンス

### 3.1 ユーザー登録

#### リクエストスキーマ (UserCreate)
```python
class UserCreate(BaseModel):
    username: str = Field(
        ...,
        min_length=3,
        max_length=50,
        regex=r'^[a-zA-Z0-9_-]+$',
        description="ユーザー名（英数字、アンダースコア、ハイフンのみ）",
        example="testuser"
    )
    email: EmailStr = Field(
        ...,
        max_length=100,
        description="メールアドレス",
        example="test@example.com"
    )
    password: str = Field(
        ...,
        min_length=8,
        max_length=128,
        description="パスワード（8文字以上）",
        example="securePassword123"
    )
    
    class Config:
        schema_extra = {
            "example": {
                "username": "testuser",
                "email": "test@example.com",
                "password": "securePassword123"
            }
        }
```

#### リクエスト例
```json
{
  "username": "testuser",
  "email": "test@example.com",
  "password": "securePassword123"
}
```

#### バリデーションルール
- **username**: 
  - 必須フィールド
  - 3-50文字
  - 英数字、アンダースコア、ハイフンのみ許可
  - 正規表現: `^[a-zA-Z0-9_-]+$`
- **email**: 
  - 必須フィールド
  - 有効なメールアドレス形式
  - 最大100文字
- **password**: 
  - 必須フィールド
  - 最小8文字、最大128文字

#### レスポンススキーマ (UserResponse)
```python
class UserResponse(BaseModel):
    id: int = Field(
        ...,
        description="ユーザーID",
        example=1
    )

        example="testuser"
    )
    email: str = Field(
        ...,
        description="メールアドレス",

    )
    created_at: str = Field(

            "example": {
                "id": 1,

                "created_at": "2025-06-23T07:30:00Z"
            }
        }
```

#### レスポンス例
```json
{
  "id": 1,
  "username": "testuser",
  "email": "test@example.com",
  "created_at": "2025-06-23T07:30:00Z"
}
```

## 4. 議事録系リクエスト・レスポンス

### 4.1 議事録生成

#### リクエストスキーマ (MinutesGenerate)
```python
class MinutesGenerate(BaseModel):
    transcript: str = Field(
        ...,
        min_length=10,
        max_length=10000,
        description="会議のトランスクリプト",
        example="今日の会議では、新しいプロジェクトについて話し合いました。参加者は田中さん、佐藤さん、鈴木さんでした。"
    )
    title: Optional[str] = Field(
        None,
        max_length=200,
        description="会議のタイトル（オプション）",
        example="週次定例会議"
    )
    
    class Config:
        schema_extra = {
            "example": {
                "transcript": "今日の会議では、新しいプロジェクトについて話し合いました。参加者は田中さん、佐藤さん、鈴木さんでした。プロジェクトの開始日は来月1日に決定しました。",
                "title": "プロジェクト企画会議"
            }
        }
```

#### リクエスト例
```json
{
  "transcript": "今日の会議では、新しいプロジェクトについて話し合いました。参加者は田中さん、佐藤さん、鈴木さんでした。プロジェクトの開始日は来月1日に決定しました。",
  "title": "プロジェクト企画会議"
}
```

#### バリデーションルール
- **transcript**: 
  - 必須フィールド
  - 最小10文字、最大10,000文字
  - 空白のみの文字列は無効
- **title**: 
  - オプションフィールド
  - 最大200文字

#### レスポンススキーマ (MinutesResponse)

#### バリデーションエラースキーマ (ValidationErrorResponse)
```python
class ValidationErrorDetail(BaseModel):
    loc: List[Union[str, int]] = Field(
        ...,
        description="エラー発生箇所",
        example=["body", "username"]
    )
    msg: str = Field(
        ...,
        description="エラーメッセージ",
        example="ensure this value has at least 3 characters"
    )
    type: str = Field(
        ...,
        description="エラータイプ",
        example="value_error.any_str.min_length"
    )

class ValidationErrorResponse(BaseModel):
    detail: List[ValidationErrorDetail] = Field(
        ...,
        description="バリデーションエラー詳細"
    )
    
    class Config:
        schema_extra = {
            "example": {
                "detail": [
                    {
                        "loc": ["body", "username"],
                        "msg": "ensure this value has at least 3 characters",
                        "type": "value_error.any_str.min_length"
                    }
                ]
            }
        }
```

#### バリデーションエラーレスポンス例
```json
{
  "detail": [

      "loc": ["body", "username"],
      "msg": "ensure this value has at least 3 characters",

    },
    {
      "loc": ["body", "email"],
      "msg": "field required",
```
